この記事の目的

このwebサイトはGitHubPagesを用いて配信されている静的サイトです． 自主ゼミ期間中，Discordでは埋もれがちな全体向けの内容をこのページを用いて，参加者らに伝達することができます． この記事を読めば記事を追加すると同時にwebサイトに反映することができるので，ぜひ習得してください．

webサイト更新までの大まかな流れ

まず，このディレクトリの構造を下に示します．

├─.devcontainer
├─.github
├─.jekyll-cache
├─.vscode
├─assets
├─bin
├─docs
├─fixtures
├─lib
├─_includes
├─_layouts
├─_sass
└─_site

本来，GitHubPagesを用いて配信されているwebサイトはgit pushを行えば自動で反映される手はずになっています． しかし，今回用いているテンプレートがGitHubPagesに対応していないため，少し面倒な手順を踏んでいます． 大まかな手順は以下の通りです．

  1. bundle exec jekyll buildを実行してローカルで.mdファイルで書かれた記事を.htmlに変換する．
  2. 変更の履歴を取るためにharry-arbrebleu/seminar-publicにgit pushする．
  3. 1で作成したファイル一式が吐き出されたディレクトリ，./_siteを別のフォルダseminar-webにコピーアンドペーストする．
  4. 作成された.htmlファイルのpathが間違っているため，修正用のpythonファイルchange.pyを実行して直す．
  5. test3をgit pushしてwebサイトが完成する．

準備

若干の環境構築が必要です．必要な環境は以下の通りです．

  • python3
  • ruby
  • git

これらの環境構築が済んでいる方はこちらへ飛んでください． 済んでいない方は適宜検索しながら環境構築してください．

記事更新の仕方

保存場所

例として，集合論ゼミの階層構造を示します． ゼミのディレクトリ名はアメリカ英語でつけてあるので，docsディレクトリの中から探してください．

set
│  main.md
│
└─corse
    ├─0
    │      main.md
    │
    └─src
            初学者のための集合論入門(演習編).pdf
            初学者のための集合論入門.pdf

ディレクトリ直下にあるmain.mdファイルはシラバスです． 必要に応じて変更してください．その際，変更日時を記事内に明記するのを忘れないようにしてください． corseディレクトリ内にある数字の書かれたフォルダはゼミの各回に対応しています． 毎回のゼミで新しくフォルダを作って，資料が散らばらないようにしてください． ゼミの各回の資料を保存できます．レジュメは.mdファイルで作成して，各回のフォルダに保存してください． srcディレクトリ内には，何度も参照したい資料をまとめて保存できます．

以下に各回のレジュメのテンプレートを示します． 適宜変更して用いてください．

title: 第n回レジュメ(nを記入)
nav_order: n(ゼミの回数を記入)
parent: (ゼミ名を記入)
grand_parent: 自主ゼミリスト

なお，レジュメは各参加者が作成しますが，参加者が直接pushすることはせず，Chefがpushしてください． これはGitHubの操作に不慣れな参加者がいることが予想されるからです． DiscordのDMなどで作成されたレジュメを受け取り，Chefがpushしてください．
